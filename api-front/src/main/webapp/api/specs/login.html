用户登录接口

  • 接口说明

    请求地址: /v1/login.json
    请求方式: POST
    登 录: 否

  • 公共参数

    请求参数
    名称 类型 是否必须 默认值 描述
    t String 可选 用户登录服务端生成的回话token 用户必须妥善保存好这个token 以后每次需要验证登录接口都必须带上
    timestamp number 必须 请求时间戳，当前时间转为毫秒的数字日期
    deviceId String 必须 设备的唯一标识
    channel String 可选 设备名称 如：xiaomi,iphone6s,iphone5等
    osv String 必须 系统版本
    apiv String 必须 api接口版本 三位数字段：1.0.0(主版本.阶段版本.修复版本)
    sign String 必须 针对所有的请求参数和url进行签名,签名规则：参考
    响应参数
    名称 类型 实例值 描述
    status String 200 响应结果(代码参考:附录一)
    type String array 返回结果中的data字段数据类型，可能值array,string,object
    msg String 登录过期重新登录 响应给用户的消息，可用于提醒用户
    data object {} 返回的结果，可能是数组，字符串，map和type字段array,string,object相对应，具体内容查看接口响应参数节点

  • 请求参数

    名称 类型 是否必须 默认值 描述
    accountName String 是 用户名
    pwd String 是 登录密码

  • 响应参数

    名称 类型 实例值 描述
    t String 登录返回token
    signKey String 0cc175b9c0f1b6a831c399e269772661 小写一个md5字符串 用户必须和token一样保存下来,该字段在后台签名时用

    "status": 0,
    "msg": "success",
    "type": "string",
    "content": "登录成功"
}
